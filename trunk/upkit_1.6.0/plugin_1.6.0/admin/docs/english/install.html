Installation

In the description below:

  • <geeklog_dir> is a folder (directory) in which Geeklogs's db-config.php resides
  • <public_html> is a folder (directory) in which lib-common.php resides
  • <admin> is a folder (directory) in which files for admin resides (Usually, the child directory of <public_html>)
  1. Back up your database by using admin's database utility or a GUI tool like phpMyAdmin, since {plugin} plugin modifies the database.
  2. Unpack a archive file of the plugin into <geeklog_dir>/plugins folder. A folder named {plugin} will be created in the plugins folder.
  3. In the <public_html>/ folder, create a folder named {plugin}, and in the <admin>/plugins/ folder, create a folder named {plugin} as well.
  4. Move to <geeklog_dir>/plugins/{plugin}/ folder and copy all the contents of the public_html folder into the <public_html>/{plugin} folder you created in the previous step. Likewise, copy all the contents of the adminfolder into the <admin>/plugins/{plugin}/ folder.
  5. Log in to your site as admin, go to the plugin editor and click on the "{plugin}" link. When installation failed, please check the error log (<geeklog_dir>/logs/error.log).

Uninstallation

  1. Log in to your site as admin, go to the plugin editor and click on the "{plugin}" link. All relevant database data will be erased.
  2. Then, remove the three folders you created when installing the plugin: <geeklog-dir>/plugins/{plugin}/, <public_html>/{plugin}/ and <admin>/plugins/{plugin}/.
