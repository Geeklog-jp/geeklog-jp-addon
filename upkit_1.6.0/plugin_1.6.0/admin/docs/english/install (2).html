概要

Geeklog {display_name} プラグインは、{display_name}の機能を提供します。

インストール

以下の説明で、

  • <geeklog_dir>は、Geeklogシステムのdb-config.phpが存在するフォルダ（ディレクトリ）
  • <public_html>は、lib-common.phpのあるフォルダ
  • <admin>は、管理用ファイルがあるフォルダ（ふつうは、<public_html>ディレクトリの直下）

を表しています。

  1. {display_name} プラグインはテーブルを追加するので、管理者メニューの「データベース」やphpMyAdminを使用して、データベースのバックアップを取ります。
  2. {display_name} プラグインのアーカイブを <geeklog_dir>/plugins フォルダ内に展開（解凍）します。plugins フォルダ内に {plugin} というフォルダができます。
  3. <public_html>/ フォルダ内に{plugin}という名前のフォルダを、また、<admin>/plugins/ フォルダ内にも {plugin}という名前のフォルダを作成します。
  4. <geeklog_dir>/plugins/{plugin}/ フォルダに移動し、public_hrmlフォルダの内容を先ほど作成した<public_html>/{plugin}/フォルダ内にコピーします。また、adminフォルダの内容を<admin>/plugins/{plugin}/フォルダ内にコピーします。
  5. 管理者としてサイトにログインし、プラグインエディタで"{plugin}"をクリックします。インストールに失敗した場合は、エラーログ(<geeklog_dir>/logs/error.log)を調べてください。

アンインストール

  1. 管理者としてサイトにログインし、「プラグイン」をクリックします。"{plugin}"を削除すると、関連するデータがデータベースから自動的に削除します。
  2. インストール時に作成した3つのフォルダ(<geeklog-dir>/plugins/{plugin}/、<public_html>/{plugin}/、<admin>/plugins/{plugin}/)を削除します。
